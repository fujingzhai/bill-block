# 更新日志

## 0.5.0

- 新增「流水检索」：在流水视图顶部「共 X 笔」左侧增加搜索按钮，点击弹出筛选面板，支持按关键词（同时匹配项目与备注）、分类（全部／指定分类／未分类）、金额区间（最低～最高）、日期区间（起～止）任意组合筛选，专为「某样东西历史上一共花了多少」这类回溯查询设计
- 筛选即时生效、弹层不关：边改条件边看流水列表实时变化，顶部笔数与合计金额、以及每日分组小计都按筛选结果同步重算；面板提供「重置」一键清空、「完成」关闭，点击外部或 Esc 同样关闭，关闭后筛选结果保留
- 存在筛选条件时搜索按钮高亮（深色实心），清晰指示当前处于筛选态；无结果时列表提示「没有符合筛选条件的记录」
- 检索仅作用于流水视图（周账、月账、统计以查看为主，不受影响）；全部为内存内本地过滤，离线即时、不产生任何额外存储

## 0.4.2

- 修复截图功能：框架改为流水列表内部滚动后，html2canvas 无法正确处理固定高度滚动容器，导致导出图整列内容重叠错乱；现在截图时临时将流水列表展开为自然高度，完整、整齐地捕获全部条目（设置、默认周/月、截图按钮等仍按原样在导出时隐藏）
- 优化截图分享效果：流水视图截图时整行「记一笔」表单与顶部视图切换标签一并隐藏，画面更干净；并修正周/月「默认时间」按钮与激活标签在导出图中因柔和阴影被 html2canvas 渲染成生硬灰块（双层背景）的问题，改为干净的单层描边，与编辑态观感一致
- 修正流水视图日期步进器的箭头不对称：日期文本居中显示，使「‹」「›」与日期两侧间距对称
- 统一周账/月账时间格内账单条目的悬停交互，与流水列表一致：去掉原先借鉴 Notion、靠 `display` 切换显隐导致的轻微重排抖动；窄格内不再为操作按钮预留空位（避免挤占项目名），改为悬停时金额平滑淡出、编辑/删除图标在原位淡入
